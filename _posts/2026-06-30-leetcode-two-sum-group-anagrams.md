## 发现问题

LeetCode 入门题看似简单，但新手在 Python 语法细节上踩坑无数——`enumerate` 的顺序、字典键值存取、可变与不可变容器的操作差异、方法调用的括号规则……每个坑都能卡半小时。这篇文章把两道经典题（1. 两数之和，49. 字母异位词分组）的 24 个常见坑一网打尽。

## 骨架认识

**哈希表（字典）把「遍历查找」从 O(n) 压到 O(1)，是两道题的共同骨架。** 但用对骨架的前提是理解 Python 的数据结构操作规则：可变对象（如列表）必须先建容器再追加，不可变对象（如整数）直接覆盖即可。

```
两数之和：边遍历边记 → 字典存 {数字: 下标} → 查补数 O(1)
字母异位词：排序定键 → 字典存 {统一标记: [原始词列表]} → 分组 O(n·k log k)
```

## 点 → 线 → 面

### 一、两数之和（Two Sum）—— 12 个坑

1. **题目到底在说什么**：给定整数数组 `nums` 和目标值 `target`，找两个数下标使它们相加等于 `target`。保证有且仅有一组解。

2. **为什么不用暴力循环**：暴力 O(n²)，用字典一边遍历一边记录「见过的数字及其下标」，查找补数变 O(1)，整体 O(n)。

3. **代码必须在 `class Solution:` 内部**，方法名必须是 `twoSum`（不是 `twoSun`）。

4. **`seen = {}` 是记事本**：格式 `{数字: 下标}`，存储之前遍历过的数字。

5. **`for i, num in enumerate(nums):`** —— `enumerate` 产 `(下标, 元素)` 元组，顺序固定（先下标后元素），不能颠倒。

6. **`complement = target - num`** ——「还缺的那个数」。

7. **`if complement in seen:` 返回 `[seen[complement], i]`** —— 补数的下标 + 当前下标。

8. **`seen[num] = i` 不能写成 `seen[num] = 1`** —— 必须记录真实下标，因为返回需要位置信息。

9. **字典里到底存了什么**：键 = 数字本身（如 `2`），值 = 它在数组中的下标（如 `0`）。

10. **最后的 `return []`** —— 题目保证一定有解，这行理论上永不执行，但保留以满足 Python 语法完整性。

11. **复杂度**：时间 O(n)，一个 for 循环；空间 O(n)，最坏存 n-1 个元素。

12. **核心思想一句话**：用字典把「回顾过去」变成「瞬间查找」，把配对问题压缩成一次遍历。

### 二、字母异位词分组（Group Anagrams）—— 12 个坑

1. **什么是字母异位词**：字母种类和个数完全相同，只是排列顺序不同。如 `"eat"`, `"tea"`, `"ate"`。

2. **解决思路**：给每个单词找「统一标记」—— 字母排序后的标准字符串（如 `"eat" → "aet"`），用字典按标记分组。

3. **`key = ''.join(sorted(word))`** —— `sorted(word)` 拆字母排序得列表，`''.join()` 粘回字符串。

4. **`if key not in groups: groups[key] = []`** —— 第一次遇到某个 key 时必须创建空列表作为容器，否则无法 `.append()`。

5. **值与两数之和的区别**：Two Sum 的值是不可变整数（直接赋值），本题的值是可变列表（先建容器再追加）。

6. **`.append(word)` 是列表自带方法**，在末尾添加元素。必须用圆括号 `()`，不能写成方括号 `[]`。

7. **`list(groups.values())`** —— 题目要求返回二维列表，`groups.values()` 是视图对象，必须 `list()` 转换。

8. **括号的三种用途**：`[]` = 创建列表或取值，`{}` = 创建字典，`()` = 调用函数或方法。`.append[word]` 是错的。

9. **为什么 `()` 有时有参数有时没有**：有参数 = 函数需要额外信息（如 `sorted(word)`），无参数 = 方法自带「自己」这个信息（如 `"abc".upper()`）。

10. **不能写成 `key(seen)` 代替 `seen.keys()`** —— `.keys()` 是字典对象的方法，全局不存在 `key()` 函数。

11. **算法复杂度**：时间 O(n·k log k)，n 个单词各排序 O(k log k)；空间 O(n·k)。

12. **常见笔误**：`''.(sorted(word))` → 点号错位；`seen[key].append[word]` → 方法调用必须用 `()`；`list(seen(value))` → 正确是 `list(seen.values())`。

### 三、两道题的共同智慧

- **哈希表核心价值**：用空间换时间，把「找」从 O(n) 降到 O(1)
- **可变 vs 不可变容器**：直接赋值还是先建容器追加，取决于存放的是单个值还是需要动态追加的一组值
- **Python 语法习惯**：`.` 访问对象的属性/方法，`()` 执行可调用对象——弄清这些符号就能少踩 80% 的坑
